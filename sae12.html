Introduction

Cette SAÉ avait pour objectif de découvrir les bases des réseaux informatiques et l’administration des services réseau sous Linux. Durant ce projet, j’ai travaillé sur la configuration réseau, le protocole DHCP ainsi que l’installation d’un serveur NGINX.

Apprentissages Critiques (AC)

  • AC11.03 — Configurer les fonctions de base du réseau local
  • AC11.04 — Maîtriser les systèmes d’exploitation
  • AC11.05 — Identifier les dysfonctionnements réseau
  • AC11.06 — Installer et configurer un poste client

Réalisation

J’ai utilisé une machine virtuelle Debian afin de réaliser plusieurs manipulations réseau. J’ai configuré l’interface réseau, effectué des tests de connexion et utilisé différentes commandes Linux pour analyser les problèmes.

Preuve 1: DHCP

Configuration du serveur DHCP

Durant cette SAÉ, j’ai configuré un serveur DHCP sur Debian afin de distribuer automatiquement des adresses IP aux différentes machines du réseau.

Après la configuration du serveur et du poste client, plusieurs tests ont été réalisés pour vérifier le bon fonctionnement du réseau.

Problème rencontré

Le poste client ne recevait plus d’adresse IP automatiquement

Pendant les tests réseau, le poste client ne recevait plus automatiquement d’adresse IP depuis le serveur DHCP.

En utilisant la commande : ip a, aucune adresse IPv4 valide n’apparaissait sur l’interface réseau du client.

Cette situation indiquait que le poste client ne communiquait plus correctement avec le serveur DHCP.

Analyse du problème

Afin de comprendre l’origine du problème, j’ai vérifié l’état du service DHCP sur le serveur Debian.

La commande : systemctl status isc-dhcp-server montrait que le service était arrêté.

Le problème provenait donc du serveur DHCP, qui ne distribuait plus les paramètres réseau aux machines clientes.

Résolution du problème

Pour corriger cette erreur, j’ai modifié les paramètres de configuration du serveur DHCP.

J’ai ensuite vérifié l’interface réseau utilisée ainsi que les informations présentes dans le fichier de configuration DHCP.

Après les modifications, le service DHCP a été redémarré afin d’appliquer la nouvelle configuration.

Résultat obtenu

Après renouvellement de la configuration réseau du poste client, une adresse IP valide a été attribuée automatiquement par le serveur DHCP.

Preuve 2 : NGINX

Découverte et analyse de NGINX

Durant cette SAÉ, j’ai découvert le serveur web NGINX sous Linux. Le serveur NGINX permet d’héberger des services web et de répondre aux requêtes HTTP des clients.

Mise en place du serveur web NGINX

J’ai installé le serveur web NGINX sur Debian puis vérifié son installation avec la commande : sudo nginx -v.

Problème rencontré avec NGINX

Après avoir arrêté le service NGINX, le serveur web ne fonctionnait plus. La commande systemctl montrait que le service était inactif.

Comme le service NGINX était arrêté, aucun processus n’écoutait sur le port 80, ce qui empêchait l’accès au site web.

Lorsque j’ai essayé d’accéder au site web avec localhost, le navigateur affichait une erreur de connexion refusée.

Analyse du problème

Pour analyser le problème, j’ai utilisé la commande : systemctl status nginx.

La commande indiquait que le service était dans l’état inactive (dead), ce qui expliquait pourquoi le site localhost était inaccessible.

Résolution du problème

Pour résoudre le problème, j’ai redémarré le service NGINX avec la commande : sudo systemctl restart nginx.

Ensuite, j’ai vérifié que le service était bien actif et en cours d’exécution.

Enfin, la commande ss -tulpn | grep nginx a confirmé que NGINX écoutait correctement sur le port 80 en TCP.

Autoévaluation
