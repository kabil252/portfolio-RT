Introduction

Cette SAÉ avait pour objectif de développer un programme Python permettant de récupérer des données OpenStreetMap grâce à l’API Overpass.

Le programme générait automatiquement des fichiers Markdown, JSON et HTML contenant des informations géographiques sur différents lieux de la ville de Caen.

Durant les tests du projet, plusieurs erreurs Python et problèmes de traitement des données ont été rencontrés puis corrigés.

Apprentissages Critiques (AC)

  • AC11.01 — Utiliser un système informatique et ses outils
  • AC11.02 — Lire, exécuter et corriger un programme
  • AC11.03 — Produire et traiter des données avec python
  • AC11.04 — Comprendre le fonctionnement d’une API

Réalisation

Durant cette SAÉ, j’ai utilisé Python afin de récupérer des données géographiques depuis l’API Overpass.

Le programme permettait ensuite de générer automatiquement des fichiers Markdown, JSON et HTML contenant différentes informations sur les cafés de la ville de Caen.

Preuve 1 : Traitement des données API

Récupération des données OpenStreetMap

Durant cette partie du projet, le programme Python utilisait l’API Overpass afin de récupérer automatiquement des données OpenStreetMap.

Les données récupérées étaient ensuite converties en fichiers JSON et HTML pour afficher les informations géographiques.

Problème rencontré

Erreur lors du traitement des données API

Pendant l’exécution du programme Python, une erreur empêchait le traitement correct des données récupérées depuis l’API.

Le script rencontrait un problème lors de la lecture des données JSON.

Analyse du problème

Après vérification du programme, j’ai constaté que le problème provenait du traitement des données JSON récupérées depuis l’API OpenStreetMap.

Le script utilisait la fonction : response.json() afin de convertir les données API.

Une erreur empêchait donc l’interprétation correcte des données JSON.

Résolution du problème

Pour corriger cette erreur, j’ai vérifié et corrigé le traitement des données JSON dans le programme Python.

Après correction, les données ont été récupérées correctement et les fichiers HTML ont été générés sans erreur.

Résultat obtenu

Après correction du programme, les fichiers HTML, JSON et Markdown ont été générés correctement.

Preuve 2 : Conversion Markdown vers HTML

Conversion automatique des fichiers HTML

Le programme Python utilisait également une fonction permettant de convertir automatiquement les fichiers Markdown en fichiers HTML.

Problème rencontré

Erreur lors de l’appel de la fonction Python

Pendant les tests du script Python, une erreur apparaissait lors de la génération du fichier HTML.

Python ne reconnaissait pas la fonction utilisée pour effectuer la conversion Markdown vers HTML.

Analyse du problème

Après analyse du script Python, j’ai constaté qu’une faute était présente dans le nom de la fonction de conversion.

La fonction appelée dans le programme ne correspondait pas au nom réel de la fonction définie dans le module Python.

Résolution du problème

Pour corriger cette erreur, j’ai remplacé le mauvais nom de fonction par le nom correct dans le script Python.

Après correction, le programme a pu générer correctement les fichiers HTML.

Résultat obtenu

Après correction du script, la conversion Markdown vers HTML fonctionnait correctement sans aucune erreur Python.
